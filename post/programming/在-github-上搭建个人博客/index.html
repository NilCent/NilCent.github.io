在 GitHub 上搭建个人博客

Jan 10, 2023
2 minute read

我曾在许多平台写过博客, 像是 CSDN, 简书, Blogger, Medium等. 但这些平台或多或少有以下几点问题:

  • 发布文章麻烦 通常文章是在本地写的, 发布文章时需要复制一遍. 如果文章里有图片, 还需要折腾图床.
  • 博客样式不可自定义 有的平台网页简陋, 还一堆广告.
  • 大陆无法访问

而自建服务器虽说可以解决以上问题, 却需要买服务器和域名.

最终我选择使用介于平台和自建博客两者的解决方案 – GitHub Pages + Actions. 每一个 GitHub 用户都可以拥有一个属于自己的网站. 只要你有 Github 账号和 git 这个软件, 就可以按照这篇文章的步骤创建个人的博客.

安装 Hugo

我个人用的系统是 Windows, 在安装 Hugo 前需要先安装包管理工具 Choco.

安装 Hugo (可以参考 Hugo 文档)

choco install hugo-extended

git submodule add https://github.com/CaiJimmy/hugo-theme-stack/ themes/hugo-theme-stack

配置远程仓库

在 GitHub 上创建两个仓库 – 私有的 myblog 库和公开的 <username>.github.io 库

创建 Personal Access Key, 注意这个 key 要有 repo 和 workflow 权限.

在 myblog 库的 secrets 中配置刚刚创建的 token

在本地 myblog 库中添加远程源

git remote add origin https://github.com/<username>/myblog.git

配置 Github Actions

使用 GitHub Actions, 可以在修改 myblog 代码的同时自动修改 <username>.github.io 库的内容, 从而实现博客的自动更新.

在本地 myblog 库中创建 .github/workflow/main.yml, 并复制下面的内容

name: CI
on:
  push:
  workflow_dispatch:
  schedule:
    # Runs everyday at 8:00 AM
    - cron: "0 0 * * *"
        
jobs:
  deploy:
    runs-on: ubuntu-22.04
    steps:
      - name: Git checkout
        uses: actions/checkout@v2
        
      - name: Update theme
        # (Optional)If you have the theme added as submodule, you can pull it and use the most updated version
        run: git submodule update --init --recursive
        
      - name: Setup hugo
        uses: peaceiris/actions-hugo@v2.6.0
        with:
          hugo-version: 'latest'
          extended: true

      - name: Build
        # remove --minify tag if you do not need it
        # docs: https://gohugo.io/hugo-pipes/minification/
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          PERSONAL_TOKEN: ${{ secrets.TOKEN }}
          EXTERNAL_REPOSITORY: <username>/<username>.github.io
          PUBLISH_BRANCH: master
          PUBLISH_DIR: ./public
          commit_message: ${{ github.event.head_commit.message }}

综上

至此, 你只需将文章和图片 push 到远程仓库就可以自动部署博客.
